# Керування ключами (симетрія)

## Генерація
- Для кожного `label` створюється запис у `keys/keystore.json`.
- Генерується **salt** (16 байт) та **master key** (32 байти) через CSPRNG.
- Для режимів шифрування ключі виводяться через HKDF:
  - `enc_key` для AES
  - `mac_key` для HMAC (у CBC)

## Ротація
- Кожна генерація нового ключа для одного `label` створює **нову версію** (v1, v2, ...).
- Рекомендовано регулярно створювати нові версії (наприклад, раз на семестр / після інцидентів),
  а старі версії зберігати лише на час, потрібний для дешифрування старих даних.

## Безпечне зберігання
- У навчальному проєкті keystore зберігається локально.
- На реальних системах ключі слід зберігати у KMS/HSM або через сховище секретів, з правами доступу 600.
