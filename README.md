# ПР №3–4. Реалізація симетричного/асиметричного шифрування 

У межах практичної роботи було розроблено програмний модуль, що реалізує симетричне, асиметричне та гібридне шифрування з використанням сучасних криптографічних алгоритмів і бібліотек. Реалізація виконана мовою Python та оформлена у вигляді консольного застосунку з можливістю вибору режимів роботи.

## Симетричне шифрування

Реалізовано симетричне шифрування на основі алгоритму AES з підтримкою режимів GCM та CBC. Для генерації ключів використовується криптографічно стійкий генератор псевдовипадкових чисел, а також випадковий salt. Передбачено механізм керування ключами з версіонуванням, що дозволяє виконувати ротацію ключів і запобігати їх тривалому повторному використанню. Контроль цілісності даних забезпечується за допомогою тега автентифікації у режимі GCM або окремого HMAC у режимі CBC.

## Асиметричне шифрування та цифровий підпис

Реалізовано асиметричний алгоритм RSA із можливістю генерації пари ключів заданого розміру. Приватний ключ зберігається у зашифрованому вигляді з використанням пароля та експортується у форматі PEM. Додано функціональність створення цифрового підпису повідомлення та перевірки його автентичності за допомогою публічного ключа. Робота алгоритму продемонстрована на прикладі підпису та перевірки текстового файлу.

## Гібридне шифрування

Інтегровано механізм гібридного шифрування, у якому симетричний ключ для AES генерується випадковим чином та шифрується асиметричним алгоритмом RSA з використанням OAEP. Зашифрований ключ передається разом із зашифрованими даними, після чого на стороні отримувача виконується відновлення симетричного ключа приватним ключем і подальша дешифрація вмісту. Такий підхід дозволяє поєднати високу продуктивність симетричних алгоритмів із безпечним обміном ключами.

## Вимірювання продуктивності

Проведено серію вимірювань часу шифрування та дешифрування для різних обсягів даних, режимів AES і розмірів ключів RSA. Результати вимірювань автоматично зберігаються у вигляді таблиці та дозволяють порівняти продуктивність різних криптографічних параметрів. Отримані дані демонструють, що симетричне шифрування є значно швидшим для великих обсягів інформації, тоді як асиметричні алгоритми доцільно застосовувати переважно для обміну ключами та створення підписів.

## Обробка виняткових ситуацій та журналювання

У застосунку передбачено обробку виняткових ситуацій, зокрема помилок завантаження ключів, спроб дешифрування пошкоджених даних та помилок автентифікації. Реалізовано базову систему журналювання, у якій фіксуються події, пов’язані зі створенням ключів, шифруванням, дешифруванням, підписом та перевіркою підпису. Це дозволяє відстежувати виконані операції та спрощує аналіз помилок.
